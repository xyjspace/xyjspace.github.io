Sharding Jdbc 初探(一) : JDBC

发表于 2018-07-04 | 分类于 并发

背景

在现实的业务场景中，随着用户量的增长，我们系统中的数据量会逐渐增多。特别是对于互联网应用，其中的代表为电商应用,，它们的数据量很大，并且增长迅速。当数据库单表的数据如果达到了千万级，数据库的吞吐能力就会受到极大的影响。在这种情况下，我们通常会做数据库分库分表，给数据库降低负载，并提高其并发能力。

Sharding-JDBC是数据库分库分表中间件，提供了数据分片，读写分离，柔性事务。并且透明化分库分表所带来的影响，让使用方尽量像使用一个数据库一样使用水平拆分之后的数据库。

在系统的学习Sharding-JDBC之前，我们首先应该回顾一下JavaEE的标准JDBC，了解一下JDBC所定义的操作数据库的规范。这样会有助于我们更加快速的入门Sharding-JDBC，并且更容易的帮助我们去理解，Sharding-JDBC是如何通过JDBC去在查询数据库的时候根据策略路由，并最终拿到我们想要的数据的。

JDBC

定义

JDBC（Java DataBase Connectivity,java数据库连接）是一种用于执行SQL语句的Java API，可以为多种关系数据库提供统一访问，它由一组用Java语言编写的类和接口组成。JDBC提供了一种基准，据此可以构建更高级的工具和接口，使数据库开发人员能够编写数据库应用程序。

JDBC是JavaEE的核心标准，Java提供接口。由各大数据库厂商提供接口的实现。

核心类

Driver.class : Driver类由数据库厂商提供。我们需要使用数据库厂商提供的类来获取数据库驱动。
Connection.Java : Connection是由Driver类生成，用于生成与数据库通讯的Statement。
Statement.java : Statement负责与数据库交互，传递sql语句，并获取执行结果。
ResultSet.java : ResultSet中包含了我们sql语句的执行结果。
